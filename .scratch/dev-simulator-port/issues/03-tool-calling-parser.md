## Question

How do we implement a robust parser that parses XML-like tags (e.g., `<add_ticket title="..." description="..." severity="..." />` or equivalent JSON blocks) from a free-text response produced by a simple LLM, and integrates them into game events?

## Answer

We have implemented a robust parser at [src/lib/parser.ts](file:///root/Mutely/game/src/lib/parser.ts).

### Parser Highlights
- **Quote and Format Flexibility**: The attribute extractor `/([\w-]+)\s*=\s*(?:"([^"]*)"|'([^']*)'|([^\s/>]+))/g` captures double-quoted, single-quoted, and unquoted parameter values (e.g. `permanent=true` or `severity="high"`).
- **Graceful Parsing of Tags**: Uses a broad regular expression to match both self-closing (`<tag ... />`) and normal matched closing tags (`<tag ...>...</tag>`).
- **Inner Content Fallback**: If an LLM provides a text block within tags (e.g. `<dev_applied ...>Some blurb text</dev_applied>`), the parser catches that content and maps it to fallback properties like `description` or `blurb`.
- **Type Conversions**: Safely parses numbers (e.g. `salary`, `velocity`, `storyPoints`, `revenueIncrease`) and booleans (`permanent`) from string values, falling back to logical defaults (e.g., standard salary of 6000, 5 story points) if properties are omitted or malformed.
- **Unique ID Generation**: Candidates generated by `dev_applied` are automatically assigned a unique ID starting with `c-`.
