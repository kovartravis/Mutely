## Question

What is the detailed TypeScript schema and initial game state values for the Dev Simulator (including developers, backlog tasks, finances, game time)? We need to define this domain model so it can be serialized for LLM prompts and client state.

## Answer

The finalized TypeScript schema for the game state is as follows:

```typescript
// A single developer on the team
type Developer = {
  id: string;
  name: string;
  role: 'frontend' | 'backend' | 'fullstack' | 'devops' | 'ml' | 'dba';
  level: 'junior' | 'mid' | 'senior' | 'staff';
  salary: number;           // monthly cost
  velocity: number;         // story points per game-day (1–10)
  morale: number;           // 0–100, affects velocity
  currentTicketId: string | null;
};

// A job applicant surfaced by the LLM — appears in /hire screen
type Candidate = {
  id: string;
  name: string;
  role: Developer['role'];
  level: Developer['level'];
  salary: number;
  velocity: number;
  blurb: string;            // one-line description generated by LLM
};

// A ticket — always created by the LLM
type Ticket = {
  id: string;
  title: string;
  description: string;
  type: 'feature' | 'bug' | 'tech_debt';
  severity: 'low' | 'medium' | 'high' | 'critical';
  storyPoints: number;
  revenueIncrease: number;  // monthly MRR delta upon completion, set by LLM
  revenuePenalty: number;   // monthly MRR penalty while open (bugs only)
  status: 'backlog' | 'todo' | 'in_progress' | 'done';
  assignedTo: string | null;  // Developer.id
  progressPoints: number;
};

// Financial snapshot
type Finances = {
  cash: number;             // starting: 500_000
  monthlyBurnRate: number;  // salaries + infrastructure
  monthlyRevenue: number;   // base MRR + completed feature revenueIncrease values
  activePenalties: number;  // sum of open bug revenuePenalty values
  runway: number;           // derived: cash / (burnRate - revenue + activePenalties)
};

// One LLM simulation loop turn — stored for curated context window
type LoopTurn = {
  turnIndex: number;
  rawResponse: string;
  parsedActions: LLMAction[];
  timestamp: number;
};

type LLMAction =
  | { type: 'add_ticket'; payload: Omit<Ticket, 'id' | 'status' | 'assignedTo' | 'progressPoints'> }
  | { type: 'add_bug_ticket'; payload: Omit<Ticket, 'id' | 'status' | 'assignedTo' | 'progressPoints'> }
  | { type: 'dev_applied'; payload: Candidate }
  | { type: 'dev_quit'; payload: { developerId: string; reason: string } }
  | { type: 'market_event'; payload: { headline: string; revenueEffect: number; cashEffect: number; permanent: boolean } };
  // permanent: true  => revenueEffect added to monthlyRevenue permanently
  // permanent: false => cashEffect is a one-time cash hit/boost only

// LLM configuration
type LLMConfig = {
  endpoint: string;
  apiKey: string;
  model: string;
  loopIntervalMs: number;   // how often to fire the LLM (real time)
  contextTurns: number;     // how many past LoopTurns to include in snapshot
  systemPrompt: string;
};

// The curated snapshot sent to the LLM each loop
type LLMSnapshot = {
  day: number;
  finances: Finances;
  teamSize: number;
  teamSummary: Array<{ name: string; role: string; level: string; workingOn: string | null }>;
  openTickets: Array<{ id: string; title: string; type: string; severity: string; status: string }>;
  recentTurns: LoopTurn[];  // last `contextTurns` entries
};

type TerminalLine = {
  type: 'input' | 'output' | 'error' | 'event';
  text: string;
  timestamp: number;
};

// Root game state
type GameState = {
  companyName: string;
  gameDayMs: number;
  currentDay: number;
  developers: Developer[];
  candidates: Candidate[];  // applicants waiting in /hire screen
  tickets: Ticket[];
  finances: Finances;
  loopHistory: LoopTurn[];
  llmConfig: LLMConfig;
  terminalHistory: TerminalLine[];
  gameStatus: 'running' | 'paused' | 'game_over';
};
```

### Key decisions locked in

- **Roles**: `frontend | backend | fullstack | devops | ml | dba`
- **Ticket types**: `feature | bug | tech_debt` — always LLM-created
- **Revenue model**: each ticket carries `revenueIncrease` (applied on `done`) and `revenuePenalty` (applied while open, bugs only). The LLM sets both values.
- **Speed control**: `gameDayMs` — real milliseconds per simulated game day
- **Dev quit**: in-progress tickets revert to `todo`
- **Market events**: `permanent: true` shifts MRR permanently; `permanent: false` is a one-time cash effect
- **LLM context**: curated `LLMSnapshot` + last N `LoopTurn` entries (N = `llmConfig.contextTurns`)
- **Initial finances**: `cash: 500_000`, `monthlyBurnRate: 0`, `monthlyRevenue: 0`, `activePenalties: 0`
